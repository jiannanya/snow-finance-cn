## 通用规则：数据采集与故障转移策略（Data Sourcing & Failover Policy）

> ⚠️ **全局强制规则：本技能所有10项能力的数据采集阶段均必须遵守以下策略，严禁跳过或以知识库估算替代真实数据。**

### 核心五条原则

| # | 原则 | 执行要求 |
|---|------|---------|
| 1 | **读手册先行** | 采集前必须参阅 [数据源手册](./references/data-sources.md)，按规定的 URL 模板、字段说明和优先级顺序操作 |
| 2 | **多源并行** | 同一关键数据项必须尝试 **≥ 2 个独立数据源**，用于交叉验证；不得单源入报告（单源须注明⚠️并建议核查）|
| 3 | **重试上限 = 2次** | 每个数据源最多重试 **2次**（含首次请求共 3 次）；超过后立即切换下一优先级，禁止继续等待或无限重试 |
| 4 | **失败换源，禁止造数** | 失败后按 P1→P2→P3→P4 切换；P1-P4 全部失败则字段标注「数据不可用」，**严禁以 LLM 知识库推算值冒充真实财务数据写入报告** |
| 5 | **来源透明标注** | 每个关键财务数字后须标注来源：`{数值}（来源：{数据源名称}，{DATE}）` |

### 故障转移标准 SOP

```text
每个字段的采集流程：
  [1] 按数据源手册确认 URL → 发起首次请求
  [2] ❌ 失败/重定向 → 重试第 1 次
  [3] ❌ 再次失败    → 重试第 2 次（上限，共 3 次请求）
  [4] ❌ 三次均失败  → 立即切换下一优先级数据源（P1→P2→P3→P4）
  [5] P4 也失败    → 字段标注「[数据不可用，P1-P4 均无响应]」→ 继续下一字段

⛔ 严禁以下行为：
  × 单源失败后继续等待或超过 2 次重试
  × 用 LLM 知识库推算值替代真实财务数据
  × 跳过数据采集步骤直接进入分析阶段
  × 报告中出现无数据来源标注的财务数字
```

### 数据来源标注规范

| 情况 | 标注格式 | 报告处理 |
|------|---------|--------|
| 单源成功 | `（来源：Yahoo Finance，单源⚠️，建议核查）` | 写入报告，注明风险 |
| 双源一致 ✅ | `（来源：同花顺×百度财经，交叉验证✅）` | 直接写入报告 |
| 双源冲突 ⚠️ | `（A={值1} vs B={值2}，冲突⚠️，优先官方公告）` | 注明差异，取官方值 |
| 所有源失败 ❌ | `[数据不可用，P1-P4 均无响应，跳过此字段]` | 留空，不估算 |

### 阶段三：交叉验证（Cross-Validation）

**核心原则：** 关键财务数据至少需要**2个独立来源一致**才可入报告。  
> ⚠️ 若 P1-P4 全部失败导致单源或无数据，须在报告中明确标注「数据不可用」，**不得以知识库估算补填**。参阅[通用数据采集策略](#通用规则数据采集与故障转移策略data-sourcing--failover-policy)。

详见 [交叉验证规则](./references/data-sources.md#交叉验证规则)。

#### 验证矩阵

| 数据项 | 主来源 | 交叉来源 | 验证方式 |
|--------|--------|---------|---------|
| 最新股价 | 同花顺 | 百度财经 | 数值相同或差异<0.1% |
| 季度营收/净利 | 同花顺财务 | 机构研报 | 数值一致，或研报预测区间包含实际值 |
| PE(TTM) vs 动态PE | 同花顺 | 百度财经 | 说明差异来源（基准期不同） |
| 存货/资产负债 | 同花顺资产负债构成 | 公司季报原文 | 数值一致 |
| 机构评级 | 同花顺机构评级 | 富途牛牛分析师 | 评级方向一致 |

#### 异常处理

- **数据不一致**：优先采信证监会指定信披平台（深交所/上交所）原始公告
- **PE矛盾**：明确区分TTM PE（历史滚动）vs 动态PE（前瞻预测）并在报告中注明
- **研报预测偏差**：记录预测值 vs 实际值的偏差幅度，评估机构预测准确性

### 阶段四：分析建模（Analysis & Modeling）

详见 [分析框架](./references/analysis-framework.md) 和 [估值模型](./references/valuation-models.md)。

#### 4.1 标准分析模块

```
[A] 公司基本面     → 主营业务、商业模式、竞争壁垒、管理层
[B] 财务健康度     → 营收增速、净利润、毛利率净利率、ROE、负债率
             盈利质量：扣非/归母比（>85% ✅ / <70% ⚠️）、CFO/净利比（>80% ✅ / <50% ⚠️）
[C] 估值水平       → PE/PB/PS/EV-EBITDA，历史分位，同业比较
[D] 行业景气度     → 供需格局、价格趋势、政策催化
[E] 资金动向       → 主力净流入、融资余额、机构龙虎榜
[F] 风险矩阵       → 行业风险、公司特有风险、财务风险
[G] 目标价估值     → 多情景PE法 + PB法 + DCF补充
[H] 信披风险扫描   → 关联交易占营收比（>20% ⚠️）、审计意见历史（非标注明）
             大股东质押比（>50% ⚠️ / >70% 🔴）、近2年减持记录、解禁计划
```

#### 4.2 LLM复杂度评分

在开始深度分析前，对任务进行复杂度评分（1-5分）：

| 维度 | 评估要素 | 权重 |
|------|---------|------|
| 行业复杂度 | 强周期/科技/消费/金融等 | 30% |
| 财务异动 | 业绩暴增/暴跌/重大事项 | 30% |
| 信息密度 | 数据源质量/研报覆盖度 | 20% |
| 时效性要求 | 季报后/重大公告后等 | 20% |

- **1-2分（简单）**：仅输出基础行情+关键财务摘要
- **3分（标准）**：完整七模块分析 + PE/PB目标价
- **4-5分（深度）**：完整分析 + DCF + 情景分析 + 同业对比 + 详细风险矩阵

### 筛选工作流（5阶段）

> 📋 **数据采集（强制规则）：** 参阅 [数据源手册](./references/data-sources.md) → 多源并行（≥2源交叉验证）→ 每源最多重试 **2次** → 失败立即切换下一优先级源（P1→P2→P3→P4）→ 全失败则字段标注「不可用」，**严禁知识库估算替代真实财务数据**。

```
[1. 需求定义] → [2. 宇宙构建] → [3. 量化筛选] → [4. 精选排序] → [5. 筛选报告]
```

#### 筛选阶段一：需求定义

| 参数 | 说明 | 示例 |
|------|------|------|
| 行业方向 | 申万行业或投资主题 | 半导体、新能源、AI算力、全市场 |
| 市值区间 | 小盘/中盘/大盘 | 50-500亿（中盘成长）|
| 投资风格 | 成长 / 价值 / 周期反转 | 成长型 |
| 时间周期 | 短期 / 中期 / 长期 | 中期（3-12个月）|
| 核心约束 | 1-3个最关键的硬性指标 | PE<30、ROE>15%、营收增速>30% |

#### 筛选阶段二：宇宙构建

> **⚠️ 候选池规模强制规则：初始候选池必须 ≥ 100 只。** 若用户指定的范围（行业/板块/市值区间）内股票总数不足 100 只，则以该范围内全部可用股票为初始池（不强制扩充至 100）；若范围充足，必须纳入至少 100 只完成后续量化筛选，不得提前人工预筛缩小池规模。

1. 确定初始股票池（行业板块 + 市值过滤），确认池规模达到上述要求
2. 排除特殊股：ST/*ST、停牌股、上市不足6个月新股
3. 排除极低流动性股（日均成交额 < 5000万元）

#### 筛选阶段三：多维量化筛选

| 维度 | 筛选指标 | 常用阈值（可调整）|
|------|---------|----------------|
| **成长性** | 营收同比增速 | >20%（成长）/ >30%（高成长）|
| **成长性** | 归母净利润同比增速 | >30% |
| **盈利质量** | ROE | >12% |
| **盈利质量** | 净利率持续性 | 改善趋势或稳定 |
| **估值** | PE(TTM) | <行业平均PE的1.2倍 |
| **估值** | PEG | <1.5 |
| **财务健康** | 资产负债率 | <65%（非金融/地产）|
| **财务健康** | 经营现金流/净利润 | >0.8 |
| **行业景气** | 申万行业景气度 | 上行期 / 高景气 |
| **资金面** | 主力近20日净流入 | >0（持续净流入）|
| **资金面** | 融资余额趋势 | 增加（做多意愿）|

#### 筛选阶段四：候选股精选与排序

1. 对量化入围股票快速核查：最新季报异常（利润突降/存货暴增/审计意见非标）、大股东减持预告、解禁期到期
2. 按 [分析框架综合评分卡](./references/analysis-framework.md) 对候选股综合排序
3. 精选 Top 5-10 候选股进入报告

### 反向筛选模式（风险规避）

反向筛选用于**识别并规避高风险股票**，适用于持仓自检或黑名单构建：

| **估值泡沫** | PE/历史分位 | >历史90%分位 且 PEG>3 |

> 📋 **数据采集（强制规则）：** 参阅 [数据源手册](./references/data-sources.md) → 多源并行（≥2源交叉验证）→ 每源最多重试 **2次** → 失败立即切换下一优先级源（P1→P2→P3→P4）→ 全失败则字段标注「不可用」，**严禁知识库估算替代真实财务数据**。

| 因子 | 核心指标 | 评分逻辑（1-10） |
|------|---------|----------------|
| **价值（Value）** | PE/PB历史5年分位（越低分越高） | <20%分位→9-10，>80%分位→1-3 |
| **成长（Growth）** | 营收+净利增速（TTM）、ROE趋势 | 增速越高且趋势上行分越高 |
| **动量（Momentum）** | 近3/6/12月涨幅相对强弱、业绩超预期幅度 | 持续正动量得高分 |
| **质量（Quality）** | ROE稳定性、扣非/归母比、CFO/净利比 | 三指标均优→9-10 |

#### 因子评分矩阵

| 因子 | 原始数据 | 行业分位 | 评分(1-10) | 权重 | 加权分 |
|------|---------|---------|-----------|------|-------|
| 价值因子 | PE={X}x，PB={Y}x | {Z}% | — | 25% | — |
| 成长因子 | 营收增速{A}%，净利增速{B}% | {Z}% | — | 30% | — |
| 动量因子 | 近3月{+C}%，近12月{+D}% | {Z}% | — | 20% | — |
| 质量因子 | ROE={E}%，CFO比={F}% | {Z}% | — | 25% | — |
| **综合因子得分** | — | — | — | 100% | **—/10** |

**得分解读：** 8-10 优质多因子共振 | 6-7 部分因子领先 | 4-5 中性 | 1-3 多因子弱势（回避）

### 工作流

> 📋 **数据采集（强制规则）：** 参阅 [数据源手册](./references/data-sources.md) → 多源并行（≥2源交叉验证）→ 每源最多重试 **2次** → 失败立即切换下一优先级源（P1→P2→P3→P4）→ 全失败则字段标注「不可用」，**严禁知识库估算替代真实财务数据**。

```
[1. 招股书解读] → [2. 财务历史重建] → [3. 估值定价] → [4. 破发风险] → [5. 解禁日历]
```

**数据来源：** 巨潮资讯（cninfo.com.cn）招股书、深交所/上交所新股信息页

#### 核心分析要点

**财务分析（上市前3年）：**
- 营收/净利趋势与CAGR
- 毛利率/净利率稳定性与同业对比
- 应收账款质量（前5大客户集中度）
- 关联交易占营收比

**发行估值合理性（参考公式）：**

$$P_{\text{合理}} = EPS_{\text{首年预测}} \times PE_{\text{同类上市公司均值}} \times (1 - \text{折价率}_{15\%-25\%})$$

**破发风险信号（任意2项触发即为高风险）：**
- 发行PE > 同业公司中位数25倍以上
- 发行价对应PB > 3x（资产密集型行业）
- 招股书披露的最新一期利润同比下滑
- 网下询价机构弃购比例较高

**解禁日历（必须标注）：**

| 解禁日期 | 解禁性质 | 预计解禁比例 | 风险等级 |
|---------|---------|------------|--------|
| 上市后6个月 | 战略配售/网下配售 | — | 🟡 中 |
| 上市后1年 | 网下配售剩余 | — | 🟡 中 |
| 上市后3年 | 控股股东+核心高管 | — | 🔴 高 |

## 核心能力十：股价K线预测（Price Scenario Forecast）

> ⚠️ **重要免责声明：本能力提供基于公开信息的情景化价格区间推演，具有高度不确定性，绝非准确预测，不构成任何投资建议。** 股价受政策突变、市场情绪、黑天鹅事件等大量不可预知因素影响，预测误差可能很大。请勿将预测结果作为买卖依据。

### 预测框架

> 📋 **数据采集（强制规则）：** 参阅 [数据源手册](./references/data-sources.md) → 多源并行（≥2源交叉验证）→ 每源最多重试 **2次** → 失败立即切换下一优先级源（P1→P2→P3→P4）→ 全失败则字段标注「不可用」，**严禁知识库估算替代真实行情/财务数据**（包括当前股价、PE、EPS 等基础输入参数）。

基于「基本面锚（估值目标价） + 技术形态（支撑阻力）+ 市场情绪」三维综合，输出**情景化价格区间**，非单点预测。

```
[当前状态评估] → [基本面锚定] → [关键支撑阻力识别] → [三情景划分] → [K线区间输出]
```

#### 时间周期与默认配置

| 周期 | 预测跨度 | 默认根数 | 适用场景 |
|------|---------|---------|---------|
| 时K（1H） | 1-3个交易日 | 24根（3天） | 短线日内参考 |
| **日K（默认）** | **1-4周** | **20根（1个月）** | **中短线参考** |
| 周K | 1-3个月 | 12根（3个月） | 中线持仓参考 |
| 月K | 6-18个月 | 12根（1年） | 趋势判断参考 |
| 年K | 2-5年 | 3根 | 长期战略参考 |

#### 三情景设定

| 情景 | 概率 | 触发条件 | 价格区间（相对基准） |
|------|------|---------|-----------------|
| 悲观 | 20% | 行业下行+负面催化 | 基准价 × 0.75-0.90 |
| **基准** | **60%** | **基本面延续+中性市场** | **基准价 × 0.95-1.15** |
| 乐观 | 20% | 超预期催化+市场情绪高涨 | 基准价 × 1.15-1.35 |

#### GBM 价格路径公式

$$S_{t+1} = S_t \cdot \exp\!\left[\left(\mu - \frac{\sigma^2}{2}\right)\Delta t + \sigma\sqrt{\Delta t}\,Z + \text{shock}_t\right]$$

其中：$\mu$ = 年化漂移率，$\sigma$ = 年化波动率，$\Delta t = 1/252$，$Z \sim \mathcal{N}(0,1)$（Box-Muller 变换），$\text{shock}_t$ = 关键事件冲击项。

**三情景标准参数（A股科技/存储板块参考，可按实际调整）：**

| 情景 | 默认权重 | 年化漂移 μ | 年化波动 σ | PRNG seed |
|------|---------|----------|-----------|-----------| 
| 🐂 乐观 | 20-25% | +80% ~ +120% | 60-70% | 123 |
| ⚖️ 基准 | 55-65% | +10% ~ +25% | 50-60% | 42 |
| 🐻 悲观 | 15-20% | -40% ~ -60% | 60-70% | 456 |

> **调参原则：** 低估值高成长标的 → 上调乐观权重、增大乐观 μ；高估值均值回归标的（当前价 ≥ 机构最高目标价）→ 上调悲观权重、减小基准 μ。

#### OHLC 日内结构生成规则

| 价格分量 | 生成公式 |
|---------|--------|
| 收盘 Close | $S_{t+1}$（GBM路径值）|
| 开盘 Open | $S_t \cdot \exp(Z_O \cdot \sigma\sqrt{\Delta t} \cdot 0.38)$ |
| 最高 High | $\max(O,C) \cdot (1 + |Z_H| \cdot \sigma\sqrt{\Delta t} \cdot 0.52)$ |
| 最低 Low  | $\min(O,C) \cdot (1 - |Z_L| \cdot \sigma\sqrt{\Delta t} \cdot 0.52)$ |

#### 核心 JavaScript 复用代码片段

```javascript
// ── Mulberry32 PRNG（固定种子，结果完全可复现）──
function mulberry32(seed) {
  return function () {
    seed |= 0; seed = (seed + 0x6D2B79F5) | 0;
    let t = Math.imul(seed ^ (seed >>> 15), 1 | seed);
    t = (t + Math.imul(t ^ (t >>> 7), 61 | t)) ^ t;
    return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
  };
}
// ── Box-Muller 正态分布对 ──
function normalPair(rand) {
  let u1; do { u1 = rand(); } while (u1 < 1e-10);
  const u2 = rand(), mag = Math.sqrt(-2 * Math.log(u1));
  return [mag * Math.cos(2 * Math.PI * u2), mag * Math.sin(2 * Math.PI * u2)];
}
// ── GBM OHLC 生成器（核心函数）──
function genOHLC({ S0, mu, sigma, seed, dates, shocks = {} }) {
  const rand = mulberry32(seed);
  const dt = 1/252, drift = (mu - 0.5*sigma*sigma)*dt, vol = sigma*Math.sqrt(dt);
  let S = S0;
  return dates.map(date => {
    const [zC, zO] = normalPair(rand), [zH, zL] = normalPair(rand);
    const close = S * Math.exp(drift + vol*zC + (shocks[date]||0));
    const open  = S * Math.exp(zO * vol * 0.38);
    const high  = Math.max(open, close) * (1 + Math.abs(zH) * vol * 0.52);
    const low   = Math.min(open, close) * (1 - Math.abs(zL) * vol * 0.52);
    S = close;
    return { date, o:+open.toFixed(2), c:+close.toFixed(2), h:+high.toFixed(2), l:+low.toFixed(2) };
  });
}
// ── 数据格式转换（⚠️ ECharts 顺序：open/close/low/high）──
function toOHLC(data) { return data.map(d => [d.o, d.c, d.l, d.h]); }
// ── 移动平均计算 ──
function calcMA(data, period) {
  const cs = data.map(d => d.c);
  return cs.map((_, i) => i < period-1 ? null
    : +(cs.slice(i-period+1, i+1).reduce((a,b)=>a+b) / period).toFixed(2));
}
```
